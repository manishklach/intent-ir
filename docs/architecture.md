# INTENT-IR Architecture

## Pipeline
1. **Agent IR (JSON)**: High-level intent generated by an LLM or orchestrator.
2. **Compiler**: Lowering to structured INTENT-IR instructions.
3. **Verifier**: Static analysis for safety, resource bounds, and correctness.
4. **Assembler**: Serialization to compact binary format.
5. **Runtime**: Deterministic execution, state tracking, and trace generation.
6. **Trace Replayer**: Debugging and auditing tool.

## Execution Model
The INTENT-IR runtime is a state-machine that processes instructions sequentially. Unlike general-purpose CPUs, it is optimized for high-latency "agent-native" operations like tool calls and inter-agent communication.

## Security Model
The Verifier ensures that agents do not exceed their allocated budgets or access resources they haven't declared. This is similar to eBPF's safety checks.
